## 1. Робота з менеджерами пакетів у Linux

Під час роботи з операційною системою Linux досить часто потрібно встановити нові програми, оновити вже наявне програмне забезпечення або видалити ті додатки, які більше не потрібні. Для виконання цих дій використовується спеціальна програма - менеджер пакетів. Вона дає можливість зручно керувати програмами як через термінал, так і за допомогою графічного інтерфейсу.

### 1.1 Поняття «пакет» та «репозиторій»

**Пакет (Package)**

Пакет - це спеціально підготовлений архівний файл, який містить програму разом із усіма необхідними компонентами для її встановлення та роботи в операційній системі.

До складу пакета входять:
- виконувані файли програми;
- бібліотеки;
- залежності;
- конфігураційні файли;
- службова інформація (версія, опис, архітектура).

Пакет є одиницею встановлення програмного забезпечення в Linux.

У різних дистрибутивах використовуються різні формати пакетів:

.deb - Debian, Ubuntu, Linux Mint

.rpm - Fedora, Rocky Linux, openSUSE

Таким чином, пакет дозволяє автоматизувати процес встановлення, оновлення та видалення програм.

**Репозиторій (Repository)**

Репозиторій - це централізоване онлайн-сховище пакетів, з якого операційна система завантажує програмне забезпечення та оновлення.

Репозиторій містить:

тисячі програм;

їх оновлення;

залежності;

інформацію про сумісність версій.

Менеджер пакетів автоматично звертається до репозиторію, перевіряє доступні версії та забезпечує коректне встановлення програм.

Репозиторії можуть бути:
- офіційними (підтримуються розробниками дистрибутива);
- додатковими;
- локальними.

### 1.2 Огляд менеджерів пакетів у Linux

У різних дистрибутивах Linux використовуються різні менеджери пакетів.

**APT (Advanced Package Tool)**

Використовується у дистрибутивах:
- Ubuntu
- Debian
- Linux Mint

Працює з пакетами формату .deb.

Основні можливості:
- пошук пакетів (apt search);
- встановлення (apt install);
- видалення (apt remove);
- перегляд інформації (apt show);
- оновлення списків пакетів (apt update);
- оновлення системи (apt upgrade);
- автоматична обробка залежностей.

Для підтвердження використання APT у системі було виконано команду:
```bash
apt --version
```

**DNF (Dandified YUM)**

Використовується у:
- Fedora
- AlmaLinux
- Rocky Linux

Працює з пакетами формату .rpm.

Основні можливості:
- встановлення та видалення програм;
- керування RPM-пакетами;
- автоматична робота із залежностями;
- оновлення системи.

**Pacman**

Використовується у:
- Arch Linux

Основні можливості:
- швидке встановлення та видалення пакетів;
- оновлення всієї системи однією командою;
- синхронізація з репозиторіями.

**Zypper**

Використовується у:
- openSUSE

Основні можливості:
- встановлення та видалення пакетів;
- керування репозиторіями;
- оновлення системи;
- перевірка залежностей.

**Пояснення**

Менеджери пакетів забезпечують централізоване та безпечне встановлення програмного забезпечення в Linux.
Пакети містять усі необхідні компоненти для роботи програм, а репозиторії є офіційними джерелами їх зберігання та оновлення.
