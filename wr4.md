## 1. Робота з менеджерами пакетів у Linux

Під час роботи з операційною системою Linux досить часто потрібно встановити нові програми, оновити вже наявне програмне забезпечення або видалити ті додатки, які більше не потрібні. Для виконання цих дій використовується спеціальна програма - менеджер пакетів. Вона дає можливість зручно керувати програмами як через термінал, так і за допомогою графічного інтерфейсу.

### 1.1 Поняття «пакет» та «репозиторій»

**Пакет (Package)**

Пакет - це спеціально підготовлений архівний файл, який містить програму разом із усіма необхідними компонентами для її встановлення та роботи в операційній системі.

До складу пакета входять:
- виконувані файли програми;
- бібліотеки;
- залежності;
- конфігураційні файли;
- службова інформація (версія, опис, архітектура).

Пакет є одиницею встановлення програмного забезпечення в Linux.

У різних дистрибутивах використовуються різні формати пакетів:

.deb - Debian, Ubuntu, Linux Mint

.rpm - Fedora, Rocky Linux, openSUSE

Таким чином, пакет дозволяє автоматизувати процес встановлення, оновлення та видалення програм.

**Репозиторій (Repository)**

Репозиторій - це централізоване онлайн-сховище пакетів, з якого операційна система завантажує програмне забезпечення та оновлення.

Репозиторій містить:
- тисячі програм;
- їх оновлення;
- залежності;
- інформацію про сумісність версій.

Менеджер пакетів автоматично звертається до репозиторію, перевіряє доступні версії та забезпечує коректне встановлення програм.

Репозиторії можуть бути:
- офіційними (підтримуються розробниками дистрибутива);
- додатковими;
- локальними.

### 1.2 Огляд менеджерів пакетів у Linux

У різних дистрибутивах Linux використовуються різні менеджери пакетів.

**APT (Advanced Package Tool)**

Використовується у дистрибутивах:
- Ubuntu
- Debian
- Linux Mint

Працює з пакетами формату .deb.

Основні можливості:
- пошук пакетів (apt search);
- встановлення (apt install);
- видалення (apt remove);
- перегляд інформації (apt show);
- оновлення списків пакетів (apt update);
- оновлення системи (apt upgrade);
- автоматична обробка залежностей.

Для підтвердження використання APT у системі було виконано команду:
```bash
apt --version
```

**DNF (Dandified YUM)**

Використовується у:
- Fedora
- AlmaLinux
- Rocky Linux

Працює з пакетами формату .rpm.

Основні можливості:
- встановлення та видалення програм;
- керування RPM-пакетами;
- автоматична робота із залежностями;
- оновлення системи.

**Pacman**

Використовується у:
- Arch Linux

Основні можливості:
- швидке встановлення та видалення пакетів;
- оновлення всієї системи однією командою;
- синхронізація з репозиторіями.

**Zypper**

Використовується у:
- openSUSE

Основні можливості:
- встановлення та видалення пакетів;
- керування репозиторіями;
- оновлення системи;
- перевірка залежностей.

**Пояснення**

Менеджери пакетів забезпечують централізоване та безпечне встановлення програмного забезпечення в Linux.
Пакети містять усі необхідні компоненти для роботи програм, а репозиторії є офіційними джерелами їх зберігання та оновлення.

### 2.3 Видалення непотрібних або застарілих пакетів

Менеджер пакетів APT дозволяє видаляти встановлені програми та очищати систему від непотрібних залежностей.

- Крок 1. Видалення пакета

Для видалення пакета htop було використано команду:
```bash
sudo apt remove htop
```
Система видалила програму та звільнила дисковий простір.

- Крок 2. Повне видалення пакета

Для повного видалення разом із конфігураційними файлами використовується команда:
```bash
sudo apt purge htop
```
Оскільки пакет вже був видалений, система повідомила, що видаляти нічого.

- Крок 3. Очищення застарілих залежностей

Для видалення непотрібних бібліотек використовується команда:
```bash
sudo apt autoremove
```
У даному випадку система повідомила, що застарілі залежності відсутні.

Команда apt remove видаляє програму,
apt purge - повністю очищає її разом із налаштуваннями,
apt autoremove - видаляє непотрібні залежності.

Таким чином, менеджер пакетів APT дозволяє підтримувати систему в чистому та оптимізованому стані.

#£# 2.4 Оновлення менеджера пакетів

Оновлення менеджера пакетів APT складається з оновлення списку доступних пакетів та встановлення нових версій програм.

- Крок 1. Оновлення списку пакетів

Для синхронізації локальної бази з репозиторіями було виконано команду:
```bash
sudo apt update
```
Система отримала актуальну інформацію про доступні версії пакетів та повідомила про наявність оновлень.

Скрін 12 - Оновлення списку пакетів (apt update)

- Крок 2. Оновлення встановлених пакетів

Для встановлення доступних оновлень використано команду:
```bash
sudo apt upgrade
```
Система визначила пакети, для яких доступні нові версії, та виконала їх оновлення.

<img width="825" height="521" alt="image" src="https://github.com/user-attachments/assets/a1bc25a3-6790-448a-82bb-851a50948d1a" />

Скрін 13 - Оновлення встановлених пакетів (apt upgrade)

- Крок 3. Повне оновлення системи

Для виконання повного оновлення з перевіркою залежностей використовується команда:
```bash
sudo apt full-upgrade
```
У даному випадку система повідомила, що додаткових оновлень немає.

<img width="792" height="352" alt="image" src="https://github.com/user-attachments/assets/81ffcbfb-5990-450e-a7f3-a8839059f5b6" />

Скрін 14 - Повне оновлення системи (apt full-upgrade)

**Пояснення**

Команда apt update оновлює інформацію про доступні пакети.

Команда apt upgrade встановлює нові версії вже встановлених програм.

Команда apt full-upgrade виконує повне оновлення з можливим оновленням залежностей.

Регулярне оновлення системи підвищує її безпеку та стабільність.
